# UI/UXレビュー（2026-03-13）

対象: OCCTO Grid Observatory ダッシュボード（デスクトップ / モバイル）

## レビュー方法
- コードベースの静的確認（コンポーネント構成・レスポンシブ条件・アクセシビリティ属性）。
- ローカル起動（`next dev`）後、Playwrightでデスクトップ/モバイルの画面キャプチャ確認。
- 単体テスト（Vitest）およびLint実行結果の確認。

## 総評
- 情報設計は「概要→詳細」導線があり、俯瞰モードも用意されているため、運用監視ダッシュボードとしての基礎は高い。
- ただし、**モバイルでの情報密度が高すぎる**ため、カード群を縦に読み下す際の負荷が高い。
- 重大な表示崩れ（レイアウト破綻）は見当たらないが、**テキスト折返し・トランケーション前提の箇所が複数あり、可読性リスク**がある。

## 指摘事項

### 1) [High] モバイル時の情報過多でスキャンしづらい
- 現状はカード/セクションの表示が多く、モバイル縦スクロールが極端に長くなる。
- 俯瞰モードはあるが、初期状態でも「重要KPI + 2〜3セクション」に畳む選択肢があると実運用で見やすい。

改善案:
- モバイル初期表示を「俯瞰モード寄り」にする（詳細セクションは折りたたみ）。
- 表示パネルの選択状態を保存（localStorage）し、再訪時の操作コストを削減。

### 2) [Medium] 主要見出し/メタ情報の折返しが窮屈
- モバイル先頭で、タイトルとメタ情報（対象日・最終取り込み）が複数行に折り返され、視線移動が増える。
- 「公開データ範囲」行も長文で、重要度に対して視覚ノイズになりやすい。

改善案:
- モバイルでメタ情報を2行→1行に要約（詳細はツールチップ/開閉）。
- タイトルを短縮版（例:「発電・送電ダッシュボード」）へ切替可能にする。

### 3) [Medium] トランケーション依存が強く、文脈が欠ける可能性
- ラベル表示に `truncate` を使う箇所が複数あり、発電所名や注記が長い場合に識別困難になりうる。

改善案:
- 省略時の `title` 属性やツールチップ表示を追加。
- モバイルのみ2行まで表示 (`line-clamp-2`) を検討。

### 4) [Low] フォント取得失敗時の体験劣化リスク
- `next/font/google` の利用により、環境によっては Google Fonts 取得失敗→フォールバックに切替となる。
- 開発時は警告で継続するが、本番ビルドで失敗要因になる可能性がある。

改善案:
- セルフホストフォント（`next/font/local`）またはフォント配信障害時の運用ガイドを用意。

## 良い点
- セクションの表示ON/OFFが可能で、利用シーン別に情報量を制御できる。
- エリア選択・日付選択・時刻スライダーが明示ラベル付きで、操作対象が分かりやすい。
- 横スクロールが必要な表を `overflow-x-auto` で囲っており、破綻回避ができている。

## 優先アクション（提案）
1. モバイル初期表示を軽量化（概要優先 + 詳細は折りたたみ）。
2. 長文ラベルの省略時補助（ツールチップ/2行表示）を追加。
3. フォント配信依存を低減（local font化）して表示差分を安定化。
